## D. 요약
- 이번 배치 핵심: **트리 d&d 이동(로컬), dir watcher(로컬), 컨텍스트 메뉴/이름변경/삭제(로컬 위주), 드래그 이미지·자동스크롤(공용), 소켓 보안/안정성(원격), 우클릭 메뉴 WM_CONTEXTMENU 정석화**.
- **원격 구현 진행(7/4 저녁)**: 삭제(`c332fb7`, `file_cmd_delete`)·이동/복사(`716ec3f`, `file_cmd_move`/`file_cmd_copy`)를 프로토콜 명령 + client SHFileOperation 위임으로 신규 구현. 앞선 배치의 "원격 삭제·이동 미구현/보류" 추정은 **해소됨**.
- **이름변경(Rename)도 원격 구현 확인**: 리스트는 초기 버전부터 socket `Rename` 으로 지원, 트리도 `message_request_rename`→socket `Rename` 로 배선되어 있음(코드 확인, 별도 커밋 아님). 주요 파일/폴더는 `is_protected` 로 정확히 검사해 이름변경 차단.
- **결론**: 트리/리스트의 파일 작업 4종(**이름변경·삭제·이동·복사**)은 로컬·원격 모두 구현 완료 상태 — 이 배치 기준으로 **원격 미구현 격차는 남아 있지 않음**. (원격은 프로토콜 명령 + client SHFileOperation/Rename 위임, 로컬은 파일시스템 API 직접 호출.)
