## 1. dir watcher — 감시 폴더 삭제 시 앱 응답없음(freeze) 데드락 수정  [Common: SCDirWatcher]
- **증상**: 탐색기 등에서 감시 중이던 폴더를 삭제/이동하면 앱이 응답없음(CPU 0). 덤프 스택 —
  `rewatch_local → CSCDirWatcher::stop → CReadDirectoryChanges::Terminate` 의 스레드 join 에서 영구 대기.
- **원인**: 감시 폴더가 삭제되면 read 가 0바이트로 완료된 뒤 재발행되지 않아 outstanding 카운트가 줄지 않고,
  이후 종료 시 워커 스레드가 끝나지 않아 join 이 무한 대기.
- **수정**: `BeginRead` 가 성공/실패(bool)를 반환. 0바이트 완료(버퍼 오버플로 또는 폴더 삭제)면 재-read 로 감시 지속,
  재-read 실패(폴더 소멸)면 그 watch 를 즉시 정리(`RemoveDeadBlock`: 카운트-1 + 목록 제거 + 파괴).
  "블록이 목록/카운트에 있음 ⟺ pending I/O 있음" 불변식을 유지해 종료 루프가 항상 정상 탈출.
- 업스트림(Jim Beveridge readdirectorychanges 최신본)도 동일 취약점 보유(success 반환값 미사용) — 그 부분을 채운 개선.

## 2. 트리 확장버튼(chevron) 라이브 갱신  [Common: SCTreeCtrl + 앱: nFTDServerDlg]
- 하위폴더 삭제 시 확장버튼 잔존 → watcher REMOVED 에서 부모의 `cChildren` 을 `has_sub_folders`(디스크) 로 재평가.
- 외부에서 폴더가 이동해 들어오면 확장버튼 생성:
  - **ADDED**: 부모 자식이 미로드면 `cChildren=1`(펼쳐진 부모면 정렬 위치에 노드 삽입).
  - **MODIFIED(디렉토리)**: 접혀서 직접 감시되지 않는 폴더는 상위 감시가 주는 MODIFIED 가 유일한 신호 → 그 노드의 `cChildren` 재평가.
- **expanded-empty**: 펼쳐진 노드에서 watcher 가 마지막 자식을 삭제하면 `TVIS_EXPANDED` 만 남아, 이후 확장버튼 클릭이
  `TVE_TOGGLE` 로 접기만 시도해 먹통(첫 확장만 되고 이후 안 됨) → 클릭 시 그 stale 상태면 대상 노드를 `refresh`(자식 재열거).

## 3. 새 폴더 생성  [앱: nFTDServerDlg]
- 트리에서 새 폴더 생성 시 노드 중복 + freeze → 생성 중 `m_dir_watcher.stop()` → 작업 → `rewatch_local()`(삭제 경로와 동일 패턴).

## 4. 리스트 폴더 전환 스크롤바 flicker  [Common: SCListCtrl]  ※ 앞선 세션에 이미 반영/푸시
- `delete_all_items` 의 clear 시점 `total=0` sync 가 세로바 잔상·가로바 썸 flicker 를 유발 → refill 이 뒤따르는 경로에선
  clear 시 sync 생략(`sync=false`), 최종 표시(`display_filelist`)가 한 번만 sync.

## 5. 탐색기식 select-on-up  [Common: SCTreeCtrl(옵션) + 앱]
- `CSCTreeCtrl::set_select_on_button_up(true)` (기본 false). 앱의 로컬/원격 트리에 적용.
  - 마우스 **DOWN 에선 선택/브라우징하지 않고**, **UP(드래그가 아닐 때)에서 선택**한다 → 드래그하려고 클릭한 폴더가
    브라우징되지 않고 기존 선택/리스트가 유지된다.
  - base `OnLButtonDown` 이 클릭의 button-up 을 내부 모달 루프에서 소비하므로, 선택은 base 리턴 직후(클릭 끝난 시점) 수행.
    드래그 시작 시 드래그 항목을 선택하지 않아 기존 선택이 유지된다.
- **재클릭 이름변경 시간창**: 선택 변경 후 2초 이내의 재클릭만 이름변경 진입(그 이후엔 선택만 유지).

## 6. 로컬 이동 "잠김"  [앱: nFTDServerDlg]
- 로컬 이동/복사(`SHFileOperation FO_MOVE`) 전에 dir watcher `stop()` → 후 `rewatch_local()`.
  소스 폴더가 현재 리스트/확장 트리 폴더로 감시 중이어도 워처 핸들이 이동을 막지 않는다(삭제 경로와 동일).

## 7. 원격 폴더리스트 크래시  [앱: nFTDServerManager]
- 빈 path 로 `get_folderlist` 요청 시 `length=0 → SendExact(0바이트) → AES(0) → CryptoPP xorbuf` assert(debug 크래시).
  빈 path 가드 추가(요청 차단 + 원인 caller 추적 로그).

## 8. 원격 이동 D:\ 루트 "될때/안될때"  [앱: nFTDServerDlg]
- **원인**: 원격(CLIENT_SIDE 소스) 분기 진입부에서 `m_transfer_from` 이 '원격 리스트의 현재 폴더'(`m_remoteCurrentPath`)로
  덮어써져, 트리 드래그(소스=특정 노드 fullpath)의 same-folder 판정이 부정확 — 원격 리스트가 마침 대상 폴더를 보고 있으면
  `from==to` 로 오판해 조용히 스킵. 그래서 "리스트가 보는 폴더에 따라" 이동이 될 때/안 될 때가 갈렸다.
- **수정**: same-folder 판정을 덮어쓴 `m_transfer_from` 이 아니라 **실제 드래그한 소스들의 부모**로 계산(로컬 분기와 동일 취지).
